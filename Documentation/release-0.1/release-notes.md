# Release 0.1 – «Первая рабочая версия API»

**Дата релиза:** 27-06-2026 
**Версия:** 0.1
**Статус:** ✅ Завершён

## Состав релиза

### Пользовательские истории (User Stories)
- [x] Как пользователь, я могу зарегистрироваться и войти в систему через API (Basic Auth)
- [x] Как пользователь, я могу сыграть в одиночную игру против компьютера через API
- [x] Как пользователь, я могу сыграть в многопользовательскую игру через API
- [x] Как пользователь, я могу получить список своих игр и их статусы через API
- [x] Как разработчик, я могу просмотреть документацию API через Swagger UI

### Технические задачи
- [x] Настроена работа с PostgreSQL через JPA
- [x] Добавлены профили `dev` (H2) и `prod` (PostgreSQL)
- [x] Создан Dockerfile и docker-compose.yml для контейнеризации
- [x] Настроен CI/CD через GitHub Actions (сборка + деплой на Cloud.ru)

### Результаты тестирования
- Ручное тестирование API через Postman: ✅ все сценарии выполнены. Один major bug (ниже).
- Автотесты: отсутствуют (планируются в релизе 1.2)

### Ограничения текущей версии
- Тестировал только позитивные E2E-сценарии. В следующем релизе осталось дополнительно протестировать возможные негативные кейсы, проверить коды ошибок. 
- Отсутствует интерфейс (Только API)

### Баги в релизе
- Ошибка при последнем ходе в заполненную доску для ничьей (Ож.результат: статус игры = DRAW. Факт.результат: Статус игры остается PLAYER_1_TURN)
